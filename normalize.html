Normalize.css

Making browsers play nice — one rule at a time.

What is Normalize.css?

Normalize.css is a small CSS file that provides cross-browser consistency in the default styling of HTML elements. Rather than stripping all default styles like a traditional CSS reset, it preserves useful defaults and only corrects the inconsistencies between browsers.

It was created by Nicolas Gallagher in 2011 and is used by major CSS frameworks including Bootstrap and Tailwind CSS.

Reset vs. Normalize

✕

CSS Reset

Strips all default browser styles. You start from a completely blank slate and must define every style yourself.

Created by Eric Meyer, 2007

✓

Normalize.css

Preserves useful defaults and only targets the styles that need fixing. You keep sensible browser behavior without rebuilding from scratch.

Created by Nicolas Gallagher, 2011

Why Does Order Matter?

When two CSS rules have the same specificity, the one that appears last in the stylesheet wins — this is the cascade.

That's why normalize.css must be linked before your own CSS files. If it came after, it could override your custom styles instead of the other way around.

✅ Correct order in <head>

<link rel="stylesheet" href="styles/normalize.css">
<link rel="stylesheet" href="styles/main.css">

What is Minification?

␣

Spaces Removed

All whitespace and indentation is stripped out.

/* */

Comments Removed

Human-readable comments are deleted from the file.

↵

Line Breaks Removed

The entire file is condensed into as few lines as possible.

The result is a .min.css file that is harder for humans to read but loads faster in the browser — reducing page load time and bandwidth usage.
